# PRD：Agent-First 跨端 AI 小说创作平台

## 1. 背景与新战略

平台最终目标已升级为：
- 作为可上架应用发布到 Apple App Store。
- 同时支持 iOS、Android、Windows、macOS。
- 支持中英文小说创作（从建书到章节连续性）。
- 开发与运维主要面向 Agent（例如 GPT-5.3-Codex），人类工程师作为质量守门和运营角色。

现状问题：
- Studio 当前更像本地 Web 工作台，尚未具备跨端产品化交付能力。
- 建书流程表达力不足，难以承接高质量创意输入。
- 开发流程以“人手工实现”为主，尚未形成 Agent-First 工程体系。

## 2. 产品愿景

把 InkOS 打造成“人人可用、Agent 可开发、全设备可创作”的小说生产平台。

愿景分三层：
1. 用户层：新手可快速写，中高阶可精细控。
2. 终端层：手机端与桌面端随时接续创作。
3. 工程层：Agent 可直接读取任务、改代码、跑测试、提交变更。

## 3. 用户与开发对象分层

## 3.1 终端用户（内容创作者）

1. 新手作者
- 不懂工作流，不懂模型配置。
- 需要“对话式建书 + 一键续写 + 可读反馈”。

2. 进阶作者
- 关注剧情结构、风格一致性、伏笔兑现率。
- 需要高级工作台和校准机制。

## 3.2 主要开发对象（核心）

1. Agent 开发者（Primary）
- 能读取任务描述并执行代码变更、测试、文档更新。
- 通过标准化接口与规范实现高频迭代。

2. 人类工程师（Secondary）
- 负责架构决策、风险评审、发布管理、线上应急。

## 4. 产品范围（升级版）

## 4.1 V1（基础产品化）

- 双模式创建（简单/高级）+ 建书摘要确认。
- 中英文创作流程统一支持。
- Agent-First 开发规范落地（任务模板、契约、自动化验收）。
- 云端项目同步基础能力（同一账号跨设备接续）。

## 4.2 V2（跨端发布）

- iOS/Android 原生壳应用上线。
- Windows/macOS 桌面应用发布。
- 通知系统与后台任务跨设备一致。

## 4.3 V3（规模化运营）

- 模板市场与团队协作。
- 多项目空间与权限体系。
- 数据分析与商业化投放能力。

## 5. 功能需求（重点）

### FR-1 双模式创作
- 快速模式：自然语言输入，系统自动整理。
- 高级模式：结构化创作工作台。

### FR-2 中英文创作全链路
- 建书、计划、写作、审计、修订均支持 `zh/en`。
- 同一书可配置主语言，支持局部切换。

### FR-3 跨设备连续创作
- 登录同一账号后可加载同一项目状态。
- 支持章节草稿与基础文件同步。

### FR-4 App 化能力
- 移动端适配（触控输入、弱网重试、后台恢复）。
- 桌面端适配（大屏编辑、快捷键、导出能力）。

### FR-5 Agent-First 开发接口
- 暴露可机读任务规范（task spec）。
- 标准化 API 契约与错误码。
- 允许 Agent 完成“需求 -> 代码 -> 测试 -> 文档”闭环。

## 6. 非功能需求

- 跨端一致性：核心流程行为一致，文案允许端差异。
- 可上架性：满足 Apple 审核要求（隐私政策、账号删除、内容申明）。
- 可观测性：所有核心行为具备追踪 ID 与审计日志。
- 可维护性：Agent 变更必须可审计、可回滚、可复现。

## 7. 关键指标（新增）

- 跨端活跃：同一用户 7 日多端登录比例。
- 接续成功率：跨设备继续写作成功率。
- 双语成功率：中英文章节生成合格率。
- Agent 交付效率：每周可合并任务数、回归通过率。
- 上架准备度：商店合规检查通过率。

## 8. 上架与合规要求（产品侧）

- 必备：
  - 隐私政策页面
  - 服务条款页面
  - 账号注销入口
  - 数据删除能力
  - 内容使用声明（AI 辅助创作）

- iOS 特别要求（规划）：
  - 登录流程稳定
  - 支付相关功能合规（若后续上线）
  - 不引导绕过平台政策

## 9. 验收标准（升级）

1. 双模式在 Web、iOS、Android 至少三端可用。
2. Windows/macOS 可安装运行并完成建书到章节续写。
3. 中英文创作链路均可完成“建书 -> 第一章 -> 第二章”。
4. Agent 按规范可独立完成一个标准迭代任务并通过 CI。
5. 上架文档与合规材料齐备，可进入提审流程。
