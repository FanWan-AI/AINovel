# 测试策略与验收标准

## 1. 测试目标

- 核心创建流程可稳定完成。
- 双模式流程行为一致且可预测。
- 章节连续性提示逻辑可复现。

## 2. 测试分层

## 2.1 单元测试

后端：
- brief schema 校验
- brief normalize 逻辑
- create-flow 状态机转移
- 错误码映射

前端：
- 模式切换组件
- 摘要编辑组件
- 补问卡片交互
- 创建按钮可用性条件

## 2.2 集成测试

- `POST /normalize -> /confirm -> /create-status` 全链路
- 创建失败重试
- 同名书籍冲突 409
- brief 持久化文件写入验证

## 2.3 端到端测试（E2E）

场景 S1：新手快速创建
1. 选择快速模式
2. 输入自然语言
3. 确认摘要
4. 创建成功并跳转书籍详情

场景 S2：专业模式创建
1. 完成四步输入
2. 预览基础文件
3. 确认创建

场景 S3：异常恢复
1. 模型异常导致 normalize 失败
2. 用户手动编辑并继续创建

## 3. 回归测试范围

每次发布必须回归：
- 现有 `BookCreate` 基础流程（旧流程）
- 章节写作接口（write/draft/revise/audit）
- 日志与 SSE 不可中断

## 4. 数据测试集建议

准备 6 组标准输入：
1. 极简一句话输入
2. 超长详细输入
3. 缺失主角信息
4. 缺失结局方向
5. 中英混合输入
6. 含敏感词输入

用于验证 normalize 稳定性与补问逻辑。

## 5. 验收标准（MVP）

功能验收：
1. 快速模式可完成建书。
2. 专业模式可完成建书。
3. 摘要确认页可编辑并生效。
4. 创建失败有明确错误提示。

质量验收：
1. 核心接口成功率 >= 98%（测试环境）。
2. 前端主流程无阻断级 bug（P0/P1）。
3. E2E 主路径通过率 100%。

## 6. 自动化建议

- CI 必跑：
  - `pnpm -r test`
  - `pnpm -r typecheck`
- 针对新接口补充 API 契约快照测试。
- nightly 跑一次创建流程 E2E。

## 7. 缺陷分级

- P0：阻断创建/数据丢失
- P1：主流程严重错误但可绕过
- P2：体验缺陷/提示不清
- P3：样式与次要问题

发布门禁：
- 不允许存在 P0/P1 未修复缺陷。

## 8. 跨端与上架测试（新增）

- iOS：
  - TestFlight 核心路径全通过
  - 隐私权限与账号删除流程可用

- Android：
  - 主流机型兼容性
  - 前后台切换恢复

- Windows/macOS：
  - 安装升级卸载
  - 本地缓存与同步恢复

## 9. Agent 回归测试（新增）

- 每个任务由 Agent 自动执行：
  - 类型检查
  - 单元测试
  - 最小 E2E
  - 文档链接检查

- 人工只对高风险变更做抽检验收。

## 10. 可控写下一章 E2E 验收流程

### 10.1 流程概述

> 覆盖链路：填干预 → 预览 plan → 写下一章 → SSE 完成

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 编辑 `story/current_focus.md` 或在 Studio 触发 draft with context | 干预内容写入 |
| 2 | `POST /api/books/:id/draft` + body `{ "context": "..." }` | `{ "status": "drafting", "bookId": "..." }` 立即返回；SSE `draft:start` 到达 |
| 3 | 等待 SSE `draft:complete` 事件 | `{ "bookId": "...", "chapterNumber": N, "title": "...", "wordCount": N }` |
| 4 | `POST /api/books/:id/write-next` | `{ "status": "writing", "bookId": "..." }` 立即返回；SSE `write:start` 到达 |
| 5 | 等待 SSE `write:complete` 事件 | `{ "bookId": "...", "chapterNumber": N, "status": "ready-for-review", ... }` |
| 6 | `GET /api/books/:id` 验证章节列表 | 新章节出现，`status` 为 `"ready-for-review"` |

### 10.2 接口字段速查

**POST `/api/books/:id/write-next`**
```json
// 请求体（可选）
{ "wordCount": 3000 }
// 响应
{ "status": "writing", "bookId": "<id>" }
// SSE start
{ "event": "write:start",    "data": { "bookId": "<id>" } }
// SSE success
{ "event": "write:complete", "data": { "bookId": "<id>", "chapterNumber": 3, "status": "ready-for-review", "title": "...", "wordCount": 2800 } }
// SSE error
{ "event": "write:error",    "data": { "bookId": "<id>", "error": "..." } }
```

**POST `/api/books/:id/draft`**（可控流程第一步：预览 plan）
```json
// 请求体（可选）
{ "context": "本章把注意力拉回师徒矛盾", "wordCount": 2500 }
// 响应
{ "status": "drafting", "bookId": "<id>" }
// SSE start
{ "event": "draft:start",    "data": { "bookId": "<id>" } }
// SSE success
{ "event": "draft:complete", "data": { "bookId": "<id>", "chapterNumber": 3, "title": "...", "wordCount": 2500 } }
// SSE error
{ "event": "draft:error",    "data": { "bookId": "<id>", "error": "..." } }
```

### 10.3 Copy-paste 级回归步骤

```bash
# 0. 启动 Studio
inkos studio

# 1. 确认书籍存在
curl -s http://localhost:8765/api/books/<BOOK_ID> | jq .nextChapter

# 2. 预览 plan（填干预 + 草稿）
curl -s -X POST http://localhost:8765/api/books/<BOOK_ID>/draft \
  -H "Content-Type: application/json" \
  -d '{"context":"本章聚焦师债主线冲突"}' | jq .
# → {"status":"drafting","bookId":"<BOOK_ID>"}

# 3. 订阅 SSE 事件（另一个终端）
curl -N http://localhost:8765/api/events
# → 等待 draft:start ... draft:complete

# 4. 写下一章
curl -s -X POST http://localhost:8765/api/books/<BOOK_ID>/write-next \
  -H "Content-Type: application/json" \
  -d '{}' | jq .
# → {"status":"writing","bookId":"<BOOK_ID>"}

# 5. 等待 SSE write:start ... write:complete

# 6. 验证新章节
curl -s http://localhost:8765/api/books/<BOOK_ID> | jq '.chapters[-1]'
# → {"number":N,"status":"ready-for-review",...}
```

### 10.4 自动化测试对应关系

| 验收链路 | 测试文件 | 测试名 |
|----------|----------|--------|
| write-next 立即返回 | `server.test.ts` | `POST write-next returns writing status immediately...` |
| write-next 传递 wordCount | `server.test.ts` | `POST write-next invokes pipeline.writeNextChapter with the given wordCount` |
| write-next 无 body | `server.test.ts` | `POST write-next works with no body...` |
| draft 立即返回 | `server.test.ts` | `POST draft (plan preview) returns drafting status immediately` |
| draft 传递 context | `server.test.ts` | `POST draft passes context (intervention text) to pipeline.writeDraft` |
| 全流程 draft → write-next | `server.test.ts` | `full controlled write-next flow: draft preview then write-next completes via pipeline` |
| write:error SSE 广播 | `server.test.ts` | `SSE write:error event is broadcast when pipeline.writeNextChapter rejects` |
| 章节状态显示正确 | `book-detail.test.ts` | `translateChapterStatus` 系列 |

### 10.5 回归定位指南

| 症状 | 可能层 | 检查点 |
|------|--------|--------|
| 写下一章按钮无响应 | UI | `BookDetail.tsx: handleWriteNext` |
| POST /write-next 返回非 200 | API | `server.ts: app.post("/api/books/:id/write-next")` |
| SSE write:complete 迟迟不来 | Pipeline | `PipelineRunner.writeNextChapter` |
| SSE write:error 来了 | Pipeline / LLM | 检查 `INKOS_LLM_API_KEY` 与模型配置 |
| 章节列表不刷新 | 前端 | `use-book-activity.ts: shouldRefetchBookView` |
| 状态显示为原始字符串 | 前端 i18n | `BookDetail.tsx: translateChapterStatus` |
