# 前端详细设计（Studio）

## 1. 目标

在保持现有 Studio 风格的基础上，新增双模式建书与章节驾驶舱基础体验。

## 2. 页面结构

新增页面建议：
- `pages/BookCreateEntry.tsx`（模式选择）
- `pages/BookCreateSimple.tsx`（快速开始）
- `pages/BookCreatePro.tsx`（专业创作）
- `pages/BookCreateReview.tsx`（建书摘要确认）
- `pages/ChapterWorkbench.tsx`（章节驾驶舱）

现有页面调整：
- `pages/BookCreate.tsx` 可改为入口代理页，逐步迁移。

## 3. 前端路由扩展

在 `App.tsx` 的 route 中新增：
- `{ page: "book-create-entry" }`
- `{ page: "book-create-simple" }`
- `{ page: "book-create-pro" }`
- `{ page: "book-create-review"; draftId: string }`
- `{ page: "workbench"; bookId: string }`

## 4. 状态管理

建议建立创建流程状态 store（可先用 React state + hook）：

`hooks/use-create-flow.ts`（新增）
- `mode`
- `rawInput`
- `normalizedBrief`
- `questions`
- `answers`
- `bookConfigDraft`
- `status`
- `error`

原则：
- 页面切换不丢数据
- 自动保存草稿到 localStorage（带过期时间）

## 5. 组件拆分建议

新增组件：
- `components/create/ModeCard.tsx`
- `components/create/BriefInputPanel.tsx`
- `components/create/BriefSummaryCard.tsx`
- `components/create/BriefQuestionCard.tsx`
- `components/create/CreateProgressBar.tsx`
- `components/workbench/ChapterGoalPanel.tsx`
- `components/workbench/ContinuityWarningsPanel.tsx`

## 6. API 接入层

新增 API 调用函数（`hooks/use-api.ts`）：
- `normalizeBrief(payload)`
- `fetchBriefQuestions(payload)`
- `confirmCreate(payload)`

交互节流：
- normalize 请求建议防抖 600ms（仅“智能整理”按钮触发，不自动高频调用）

## 7. 关键交互流程

### 7.1 快速模式

1. 输入自然语言
2. 点击“整理成创作摘要”
3. 展示摘要并可编辑
4. 回答补问（如存在）
5. 确认并创建

### 7.2 专业模式

1. 分步填写创作蓝图
2. 分步填写世界人物与剧情承诺
3. AI 生成基础文件草案
4. 用户逐项确认/锁定
5. 创建书籍

## 8. UI/UX 细节约束

- 所有高级字段默认折叠，点击“展开高级选项”后显示。
- 错误提示以用户可理解语言展示，不暴露底层堆栈。
- 输入内容长度限制有“软提醒”而非强拦截（例如 8k 字提示）。
- 建书中状态必须可见，避免“点击后无反馈”。

## 9. 章节驾驶舱（基础版）UI

三区布局：
- 左栏：章节上下文（上一章摘要、钩子、角色状态）
- 中栏：本章目标（推进线、禁区、情绪）
- 右栏：生成控制（风格滑杆与预设按钮）

底部 tabs：
- 草稿
- 审核
- 连续性提示
- 下一章建议

## 10. 埋点设计

建议埋点函数：
- `trackCreateModeSelected(mode)`
- `trackBriefNormalized(success, latency)`
- `trackBriefConfirmed(mode)`
- `trackBookCreateResult(success, errorCode?)`
- `trackChapterWorkbenchGenerate(bookId, chapterNo)`

## 11. 前端测试要求

- 组件单测：
  - 模式切换
  - 摘要编辑与保存
  - 补问流程

- 页面集成测试：
  - 快速模式从输入到创建成功
  - 专业模式必填项校验
  - 创建失败重试

## 12. 跨端 UI 适配规范（新增）

- iOS/Android：
  - 优先单栏流程
  - 大输入框键盘避让
  - 弱网重试提示

- Windows/macOS：
  - 多栏布局与快捷键
  - 批量文件操作入口
  - 长文本编辑性能优化

- 统一约束：
  - 所有核心页面支持中英文 UI
  - 保持一致的错误提示语义与状态机

## 13. Agent 面向开发要求（新增）

- 页面状态与 API 错误码必须可机读，便于 Agent 自动回归。
- 前端埋点事件字段保持稳定，避免 Agent 自动化脚本失效。
- 新页面必须提供最小端到端测试脚本，供 Agent 执行。
