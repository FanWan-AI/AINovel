# 运维部署与可观测性（全章节操作）

## 1. 标准观测入口

- SSE 实时流：`GET /api/events`
- 运行中心历史流：`GET /api/runtime/events`
- 章节 run 列表：`GET /api/books/:id/chapter-runs`
- 单次 run 状态：`GET /api/books/:id/chapter-runs/:runId`
- 单次 run 事件：`GET /api/books/:id/chapter-runs/:runId/events`
- 单次 run diff：`GET /api/books/:id/chapter-runs/:runId/diff`

## 2. 全章节操作验收流程（rewrite / anti-detect）

1. `POST /api/books/:id/revise/:chapter` 提交建议（`mode` 支持 `"rewrite"`、`"anti-detect"`）。
2. 记录响应里的 `runId`。
3. 使用 `GET /api/books/:id/chapter-runs/:runId/events` 验证事件流：`start -> success/fail`。
4. 使用 `GET /api/books/:id/chapter-runs/:runId` 验证最终状态字段：`status`、`decision`、`unchangedReason`、`error`。
5. 使用 `GET /api/books/:id/chapter-runs/:runId/diff` 回看 `beforeContent`、`afterContent`、`briefTrace`。

## 3. 字段对齐清单（当前实现）

### 3.1 提交响应（`POST /api/books/:id/revise/:chapter`）

```json
{
  "status": "revising",
  "runId": "uuid",
  "bookId": "demo-book",
  "chapter": 3,
  "mode": "rewrite",
  "appliedBrief": "保留人物动机"
}
```

### 3.2 run 状态（`GET /api/books/:id/chapter-runs/:runId`）

```json
{
  "runId": "uuid",
  "bookId": "demo-book",
  "chapter": 3,
  "actionType": "rewrite",
  "status": "succeeded",
  "decision": "applied",
  "appliedBrief": "保留人物动机",
  "unchangedReason": null,
  "startedAt": "2026-01-01T00:00:00.000Z",
  "finishedAt": "2026-01-01T00:00:01.000Z",
  "error": null
}
```

### 3.3 run 事件（`GET /api/books/:id/chapter-runs/:runId/events`）

```json
{
  "runId": "uuid",
  "events": [
    { "index": 0, "runId": "uuid", "type": "start", "status": "running", "timestamp": "..." },
    { "index": 1, "runId": "uuid", "type": "success", "status": "succeeded", "timestamp": "..." }
  ]
}
```

### 3.4 run diff（`GET /api/books/:id/chapter-runs/:runId/diff`）

```json
{
  "runId": "uuid",
  "status": "succeeded",
  "decision": "unchanged",
  "unchangedReason": "No revisions were applied.",
  "beforeContent": "# 第3章\\n...",
  "afterContent": "# 第3章\\n...",
  "briefTrace": [{ "text": "降低AI痕迹", "matched": false }]
}
```

## 4. 故障分类与排障动作

| 分类 | 用户可见提示 | 快速定位 |
|------|--------------|----------|
| 模型空返回 | `PLAN_LOW_CONFIDENCE`（409） | 查 `POST /api/books/:id/next-plan` 响应体 + `plan:fail` |
| 质量门槛回滚 | `rewrite:fail`（运行中心错误事件） | 查 `/api/runtime/events?bookId=<id>` 与 run `error` |
| 事件漏接 | 前端无新事件但 run 已结束 | 用 `/chapter-runs/:runId/events` 回放 |
| 落盘失败 | run `status="failed"` 且 `error` 含写盘异常 | 查 `/chapter-runs/:runId` + 系统目录权限 |
| 权限/路径错误 | `INVALID_BOOK_ID`（400）或章节 404/500 | 直接查接口响应 `error.code`/`error` |

## 5. 一键验收命令（可复制）

```bash
BOOK_ID="demo-book" CHAPTER=3 BASE_URL="http://localhost:4569" bash -lc '
set -euo pipefail
for MODE in rewrite anti-detect; do
  RESP=$(curl -s -X POST "$BASE_URL/api/books/$BOOK_ID/revise/$CHAPTER" \
    -H "Content-Type: application/json" \
    -d "{\"mode\":\"$MODE\",\"brief\":\"验收-$MODE\"}")
  RUN_ID=$(echo "$RESP" | jq -r ".runId")
  echo "$RESP" | jq .

  for _ in $(seq 1 30); do
    STATUS=$(curl -s "$BASE_URL/api/books/$BOOK_ID/chapter-runs/$RUN_ID" | jq -r ".status")
    [ "$STATUS" != "running" ] && break
    sleep 1
  done

  curl -s "$BASE_URL/api/books/$BOOK_ID/chapter-runs/$RUN_ID/events" | jq .
  curl -s "$BASE_URL/api/books/$BOOK_ID/chapter-runs/$RUN_ID" | jq .
  curl -s "$BASE_URL/api/books/$BOOK_ID/chapter-runs/$RUN_ID/diff" | jq "{runId,status,decision,unchangedReason,beforeContent,afterContent,briefTrace}"
done
'
```
