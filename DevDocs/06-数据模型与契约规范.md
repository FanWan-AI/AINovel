# 数据模型与契约规范

## 1. 设计原则

- 前后端共享契约，避免类型漂移。
- 兼容现有 `book.json` 与章节索引结构。
- 新增数据不破坏旧流程。

## 2. Book 配置（沿用）

参考 `packages/core/src/models/book.ts`：
- `id`
- `title`
- `platform`
- `genre`
- `status`
- `targetChapters`
- `chapterWordCount`
- `language`
- `createdAt`
- `updatedAt`

## 5. API 契约建议

统一返回格式：
```json
{
  "ok": true,
  "data": {}
}
```

错误：
```json
{
  "ok": false,
  "error": {
    "code": "XXX",
    "message": "用户可读描述"
  }
}
```

## 6. 文件落盘规范

目录：
- `books/<bookId>/story/brief/raw_brief.md`
- `books/<bookId>/story/brief/normalized_brief.json`
- `books/<bookId>/story/brief/brief_versions.jsonl`

规范：
- UTF-8 编码
- json 文件保证可 parse
- jsonl 每行一个对象，追加写入

## 7. 版本与兼容

建议引入 schema version：
```json
{
  "schemaVersion": "1.0.0",
  "brief": {}
}
```

兼容策略：
- 读取时允许缺省字段，用默认值补齐。
- 写入时总是写当前版本，必要时带 migration 函数。

## 8. 字段约束建议

- `title`: 1-80 字符
- `coreGenres`: 1-8 项
- `positioning`: <= 200 字
- `rawInput`: <= 12000 字符
- `forbiddenPatterns`: <= 20 项

## 9. 安全约束

- `bookId` 必须走现有安全校验（禁止路径穿越）。
- 所有输入在服务端做 zod 校验。
- 日志中避免记录完整 rawInput，记录摘要 hash 即可。
