# 风险与安全方案

## 1. 目标

在保障用户创作自由和平台可用性的同时，控制安全风险、数据风险与内容风险。

## 2. 核心风险清单

1. 密钥泄露风险
- API Key 被写入配置文件、日志或前端状态。

2. 路径安全风险
- bookId/genreId 等路径参数导致目录穿越。

3. 内容风险
- 模型输出触发提供方审查拒绝（403）。
- 输出内容违反平台政策造成不可发布。

4. 稳定性风险
- 长上下文导致内存峰值过高、进程被杀。

5. 质量风险
- 创建流程信息不完整，导致后续章节割裂。

## 3. 安全控制措施

## 3.1 密钥管理

- API Key 只从环境变量读取。
- 不回显到 UI，不落盘到 `inkos.json`。
- 日志中对 key 做掩码处理。

## 3.2 输入校验

- 所有新接口必须 zod 校验。
- `bookId`、`genreId` 延续现有安全校验策略。
- 对长文本输入做上限保护与告警。

## 3.3 错误与降级

- 403/401 返回用户可读提示 + 排障建议。
- 模型不可用时可切换降级模型（配置层）。
- normalize 失败允许手动编辑继续创建。

## 3.4 资源保护

- 限制并发创建任务数。
- 限制长上下文长度与 maxTokens。
- 对高风险模型调用增加超时与重试上限。

## 4. 内容风险治理（产品层）

- 提供“风格禁区”与“禁止模式”输入项。
- 提供章节审稿和连续性告警。
- 提供敏感词检测入口与报告。

## 5. 审计与追踪

建议记录：
- 谁在何时确认了哪版 brief
- 哪次生成使用了哪个模型
- 哪次失败是什么错误码

用于问题复盘与责任界定。

## 6. 应急响应流程

事件级别：
- SEV-1：核心创建流程不可用
- SEV-2：部分模型不可用或高失败率
- SEV-3：局部功能异常

响应流程：
1. 触发告警
2. 值班确认影响面
3. 开启降级策略/回滚开关
4. 发布修复
5. 复盘并更新文档

## 7. 合规建议

- 明示“AI 生成辅助”的产品定位。
- 用户作品数据默认本地项目目录，提供删除能力。
- 若未来云化，需补充数据合规与权限体系文档。

## 8. 安全验收清单

1. API Key 未在前端可见。
2. 路径参数穿越测试通过。
3. 敏感操作有权限与确认（至少二次确认）。
4. 错误日志不含原始敏感信息。
5. 有完整故障应急流程。

## 9. 上架与多端风险补充（新增）

- App Store 审核风险：
  - 缺少账号删除入口
  - 隐私声明与实际不一致
  - 内容策略说明不足

- 多端同步风险：
  - 不同设备并发编辑造成覆盖
  - 离线回写导致旧版本覆盖新版本

缓解：
- 强制版本号与冲突检测
- 高风险文件启用合并确认
- 提交前后进行服务器端一致性校验

## 10. Agent 风险补充（新增）

- 风险：Agent 误删或误改关键文件  
缓解：高风险路径写保护 + 审批流程。

- 风险：Agent 生成不符合政策内容  
缓解：策略审查模型 + 输出前规则检测。
