# 开发计划与 Roadmap（跨端 + Agent-First）

## 1. 目标

以“可上架、可跨端、可被 Agent 持续开发”为核心目标推进迭代。

## 2. 阶段规划

## Phase 0（2 周）：工程底座升级

目标：
- 建立 Agent-First 开发规范与任务模板。
- 引入 `v2` 接口与 feature flag。

任务：
1. 定义 task spec（目标、范围、验收）。
2. 新增 API 契约版本化（`/api/v2`）。
3. 建立 CI 门禁（类型、测试、文档一致性）。

## Phase 1（3 周）：Web 双模式 + 中英文链路

目标：
- Web 端完成双模式创建与中英文创作主链路。

任务：
1. 快速/高级模式创建页上线。
2. brief 规范化 + 摘要确认流程上线。
3. `zh/en` 全链路参数传递打通（create/plan/compose/audit/revise）。

验收：
- 中英文各完成 1 本书从建书到第 2 章。

## Phase 2（4 周）：跨设备同步能力

目标：
- 同一账号在多设备可继续创作。

任务：
1. 建立 Project Sync Service。
2. 关键文件同步与冲突提示。
3. 登录态与设备会话管理。

验收：
- 手机创建、桌面续写、另一设备审稿可串联完成。

## Phase 3（4 周）：移动端应用（iOS/Android）

目标：
- 上线可提审版本。

任务：
1. 移动端壳应用（React Native/Expo）接入。
2. 移动端核心流程：创建、续写、审稿、导出。
3. 合规页面：隐私政策、服务条款、账号删除。

验收：
- iOS TestFlight 与 Android 内测包稳定运行。

## Phase 4（3 周）：桌面端应用（Windows/macOS）

目标：
- 桌面安装包可发布。

任务：
1. 桌面壳应用（Tauri/Electron）接入。
2. 文件导入导出与快捷操作优化。
3. 崩溃恢复与日志采集。

验收：
- Windows/macOS 安装运行并通过核心流程回归。

## Phase 5（持续）：Agent Control Plane 与规模化

目标：
- 让 Agent 成为主要开发执行者。

任务：
1. 上线 Agent 任务执行接口。
2. 建立自动化任务验收流水线。
3. 建立高风险操作审批机制。

## 3. 里程碑

- M1：Web 双模式 + 中英文上线
- M2：跨设备同步上线
- M3：iOS/Android 内测上线
- M4：Windows/macOS 发布版上线
- M5：Agent-First 研发流水线稳定运行

## 4. 资源建议

- 平台后端：2 人
- Web 前端：1-2 人
- 移动端：1-2 人
- 桌面端：1 人
- QA：1 人
- SRE/运维：1 人（可兼职）

## 5. 风险与应对

- 风险：上架审核失败  
应对：提前完成合规材料、内容策略、账号删除链路。

- 风险：多端同步冲突  
应对：版本快照 + 冲突可视化 + 手动合并。

- 风险：Agent 提交质量波动  
应对：标准任务模板 + 自动测试 + 人工审批。

## 6. 优先级（MoSCoW）

Must：
- Web 双模式与中英文链路
- 同步基础能力
- iOS 上架准备
- Agent 任务规范

Should：
- Android/桌面端同周期推进
- 章节驾驶舱高级控制

Could：
- 模板市场与协作功能

Won't（当前周期）：
- 复杂商业结算与广告系统
