## 对话创建新书（Chat-to-Book）完整实施方案

### 整体架构

```
用户聊天 → [意图路由] → create_new_book 通道
    → [向导服务] → LLM 生成草案
    → SSE 流 → BookCreationDraftCard 卡片
    → 用户微调（0-N轮）
    → 用户"确认创建"
    → POST /api/v2/books/create/confirm
    → 跳转书籍页
```

### 第二步：向导服务

**新文件：`packages/studio/src/api/services/chat-to-book-wizard-service.ts`**

```typescript
export interface WizardTurnInput {
  readonly sessionId: string;
  readonly userText: string;
  readonly previousDraft?: BookCreationDraftPayload | null;
  readonly llmCall: (prompt: string) => Promise<string>; // 注入依赖
}

export interface WizardTurnOutput {
  readonly responseText: string;       // agent 回复给用户的自然语言
  readonly updatedDraft: BookCreationDraftPayload;
  readonly readyToConfirm: boolean;    // 是否可以展示确认按钮
}

export async function processWizardTurn(input: WizardTurnInput): Promise<WizardTurnOutput>
```

**LLM 提示策略（分两个阶段）：**

阶段 A（`stage: "gathering"`，首轮或无草稿）：
```
系统提示：你是一个网文策划助理。用户想创建一本新书。
你的任务：
1. 根据用户提供的只言片语，主动推断+填充完整设定
2. 不要追问超过1个问题
3. 直接输出完整书籍设定方案（JSON格式）：
   { title, genre, audience, protagonist, coreConflict, femaleLeads, firstVolumePlan, styleRules }
4. 同时输出一段自然语言总结给用户看

用户说：{userText}
```

阶段 B（`stage: "draft_ready"`，已有草稿，用户在微调）：
```
系统提示：当前书籍草案如下：
{JSON.stringify(previousDraft)}

用户想修改：{userText}

请：
1. 根据用户意见更新草案（仅修改用户提到的字段）
2. 返回更新后的完整草案 JSON
3. 给出一句确认语，告知用户改了哪里
```

**确认检测**（无需 LLM）：
```typescript
const CONFIRM_PATTERNS = /确认|创建|开始|好的|就这样|没问题|ok|yes|行|可以/iu;
const CANCEL_PATTERNS = /取消|算了|不要|重来/iu;

export function detectConfirmation(text: string): "confirm" | "cancel" | "refine"

const CHAT_TO_BOOK_PATTERN = /(?:想|要|帮我|来)(?:写|创作|策划).{0,8}(?:一本|本|部).{0,6}(?:书|小说)|(?:新书|新小说).{0,10}(?:策划|创建)|开.{0,4}(?:新书|新坑)/iu;

if (scopeBookIds.length === 0 && CHAT_TO_BOOK_PATTERN.test(userRequest)) {

```

**`handleChatToBookWizardTurn` 实现：**

```typescript
async function handleChatToBookWizardTurn(params: {
  sessionId: string;
  userText: string;
  stream: SSEStreamingApi;
  artifactService: AssistantArtifactServiceInstance;
  llmProvider: LLMProvider;
  confirmCreateBook: (req: ConfirmCreateRequest) => Promise<{ bookId: string }>;
}): Promise<void>
```

内部流程：
1. 加载 session artifact `book_creation_draft`（若有）
2. 检测是否是"确认创建"指令
   - 若是 → 直接调 `confirmCreateBook()` → 返回 `{ type: "book_created", bookId }` 事件
   - 若不是 → 调 `processWizardTurn()` 生成/更新草案
3. 保存更新后的 artifact
4. 通过 SSE 推送：
   - `assistant:progress` 含 `{ type: "tool_call", tool: "draft_book_config" }`
   - `assistant:done` 含 `{ ok: true, response: responseText, cards: [{ type: "book_creation_draft", payload: updatedDraft }] }`

### 第六步：session 草案持久化

将 `BookCreationDraftPayload` 存到 session memory（复用已有的 `AssistantMemoryService`）：

```
key: "book_creation_wizard_draft"
layer: "session"
expiresAt: 当前时间 + 2小时（防止草案无限残留）
```

这样即使用户刷新页面，重新进入对话后 agent 能"续上"上次的草案。
