# 运行手册与故障演练（U12）

## 1. 目的与边界

- 把 Studio 常见故障排查沉淀为可复制执行的 runbook。
- 用可回归的 chaos 演练覆盖：模型超时、task snapshot 损坏、预算耗尽中断。
- 演练命令默认只使用测试沙箱 / 临时目录，不污染真实项目数据。

关联设计：[`17-顶级小说创作智能体设计与实现方案.md`](./17-顶级小说创作智能体设计与实现方案.md)

## 2. 演练前准备（隔离真实数据）

先在仓库根目录执行：

```bash
pnpm install --frozen-lockfile
pnpm --filter @actalk/inkos-core build
```

只跑 chaos 演练（测试内部使用 `mkdtemp` 临时目录）：

```bash
pnpm --filter @actalk/inkos-studio test -- src/api/server.test.ts -t chaos
```

启动本地 Studio API（用于 runbook 中的在线排障命令）：

```bash
pnpm --filter @actalk/inkos-studio dev:api
```

另开一个终端，统一设置：

```bash
export BASE_URL="http://localhost:4569"
export BOOK_ID="demo-book"
export CHAPTER="3"
```

## 3. 标准观测入口

- 运行中心事件：`GET /api/runtime/events`
- 助手任务历史：`GET /api/assistant/tasks`
- 助手单任务快照：`GET /api/assistant/tasks/:taskId`
- 章节 run 列表：`GET /api/books/:id/chapter-runs`
- 单 run 状态：`GET /api/books/:id/chapter-runs/:runId`
- 单 run 事件：`GET /api/books/:id/chapter-runs/:runId/events`
- 单 run diff：`GET /api/books/:id/chapter-runs/:runId/diff`

统一巡检命令：

```bash
curl -s "$BASE_URL/api/runtime/events?limit=20" | jq .
curl -s "$BASE_URL/api/assistant/tasks?limit=10" | jq .
curl -s "$BASE_URL/api/books/$BOOK_ID/chapter-runs?chapter=$CHAPTER&limit=10" | jq .
```

## 4. 常见故障与恢复步骤

### 4.1 模型超时 / 任务长时间停在 running

现象：

- `assistant/tasks/:taskId` 长时间为 `running`
- 或 run 事件只有 `start`，迟迟没有 `success` / `fail`

定位：

```bash
TASK_ID="asst_t_chaos_timeout_001"
curl -s "$BASE_URL/api/assistant/tasks/$TASK_ID" | jq .
curl -s "$BASE_URL/api/runtime/events?event=assistant:done&limit=20" | jq .
curl -s "$BASE_URL/api/books/$BOOK_ID/chapter-runs?chapter=$CHAPTER&limit=10" | jq .
```

恢复步骤：

1. 先确认是否存在对应 `runId` 且 `status=running`。
2. 如确认是模型侧卡死，重新发起一次定点修订，避免继续等待旧任务：

```bash
curl -s -X POST "$BASE_URL/api/books/$BOOK_ID/revise/$CHAPTER" \
  -H "Content-Type: application/json" \
  -d '{"mode":"spot-fix","brief":"重试：仅修复阻断问题"}' | jq .
```

3. 如需验证系统是否仍可工作，执行 chaos 回归：

```bash
pnpm --filter @actalk/inkos-studio test -- src/api/server.test.ts -t "chaos.*time"
```

### 4.2 助手任务快照损坏 / 历史记录打不开

现象：

- `GET /api/assistant/tasks` 返回空历史或服务重启后看不到旧任务
- 怀疑 `.inkos/assistant-task-snapshots.json` 被截断或写坏

定位：

```bash
curl -s "$BASE_URL/api/assistant/tasks?limit=20" | jq .
```

恢复步骤：

1. 不直接在真实项目目录手改快照文件，先跑隔离演练确认服务能自动兜底：

```bash
pnpm --filter @actalk/inkos-studio test -- src/api/server.test.ts -t "chaos.*snapshot"
```

2. 若线上 / 本地实例重启后任务历史仍为空，以新任务触发一次重新落盘：

```bash
curl -s -X POST "$BASE_URL/api/assistant/execute" \
  -H "Content-Type: application/json" \
  -d '{
    "taskId":"runbook_snapshot_recover_001",
    "sessionId":"runbook_snapshot_recover_session_001",
    "approved":false,
    "plan":[
      {"stepId":"s1","action":"audit","bookId":"demo-book","chapter":3},
      {"stepId":"s2","action":"revise","mode":"rewrite","bookId":"demo-book","chapter":3},
      {"stepId":"s3","action":"re-audit","bookId":"demo-book","chapter":3}
    ]
  }' | jq .
```

3. 再次确认任务历史已恢复为可解析 JSON：

```bash
curl -s "$BASE_URL/api/assistant/tasks?limit=20" | jq .
```

### 4.3 评分异常低 / optimize 多轮后仍无法达标

现象：

- `overallScore` 持续低于预期
- `blockingIssues` 持续非空
- optimize 返回 `needs_confirmation`

定位：

```bash
curl -s -X POST "$BASE_URL/api/assistant/evaluate" \
  -H "Content-Type: application/json" \
  -d "{\"scope\":{\"type\":\"chapter\",\"bookId\":\"$BOOK_ID\",\"chapter\":$CHAPTER}}" | jq .
```

恢复步骤：

1. 先读 `blockingIssues`，确认是失败 run 还是 unchanged run 拉低分数。
2. 若是单章问题，执行有限次 optimize 并保留 `retryContext`：

```bash
curl -s -X POST "$BASE_URL/api/assistant/optimize" \
  -H "Content-Type: application/json" \
  -d "{
    \"taskId\":\"runbook_optimize_low_score_001\",
    \"sessionId\":\"runbook_optimize_low_score_session_001\",
    \"scope\":{\"type\":\"chapter\",\"bookId\":\"$BOOK_ID\",\"chapter\":$CHAPTER},
    \"targetScore\":85,
    \"maxIterations\":2
  }" | jq .
```

3. 如果返回 `needs_confirmation`，转人工确认，不要无限重试。

### 4.4 预算耗尽 / 策略门禁中断

现象：

- `POST /api/assistant/execute` 返回 `ASSISTANT_EXECUTE_POLICY_BLOCKED`
- 运行中心出现 `assistant:budget:warning`

定位：

```bash
curl -s -X POST "$BASE_URL/api/assistant/policy/check" \
  -H "Content-Type: application/json" \
  -d '{
    "approved":true,
    "budget":{"spent":1200,"limit":1000,"currency":"tokens"},
    "plan":[
      {"stepId":"s1","action":"audit","bookId":"demo-book","chapter":3},
      {"stepId":"s2","action":"revise","mode":"spot-fix","bookId":"demo-book","chapter":3},
      {"stepId":"s3","action":"re-audit","bookId":"demo-book","chapter":3}
    ]
  }' | jq .

curl -s "$BASE_URL/api/runtime/events?event=assistant:budget:warning&limit=20" | jq .
```

恢复步骤：

1. 降低本次执行范围（例如只保留 audit 或单章修复）。
2. 预算补充后重新执行。
3. 用 chaos 回归确认预算门禁会在写入前中断：

```bash
pnpm --filter @actalk/inkos-studio test -- src/api/server.test.ts -t "chaos.*budget"
```

### 4.5 数据恢复（误删章节 / run）

现象：

- 误删章节或误删 chapter run，需要恢复

恢复步骤：

1. 先做预览：

```bash
curl -s -X POST "$BASE_URL/api/assistant/delete/preview" \
  -H "Content-Type: application/json" \
  -d "{\"target\":\"chapter\",\"bookId\":\"$BOOK_ID\",\"chapter\":$CHAPTER}" | jq .
```

2. 确认后执行删除，记录返回的 `restoreId`：

```bash
curl -s -X POST "$BASE_URL/api/assistant/delete/execute" \
  -H "Content-Type: application/json" \
  -d "{\"target\":\"chapter\",\"bookId\":\"$BOOK_ID\",\"chapter\":$CHAPTER}" | jq .
```

3. 用 `restoreId` 恢复：

```bash
RESTORE_ID="asst_restore_xxx"
curl -s -X POST "$BASE_URL/api/assistant/delete/restore" \
  -H "Content-Type: application/json" \
  -d "{\"restoreId\":\"$RESTORE_ID\"}" | jq .
```

### 4.6 回滚（重写前回退到上一章状态）

适用场景：

- 当前章节已不可用，需要基于上一章状态重新生成

执行：

```bash
curl -s -X POST "$BASE_URL/api/books/$BOOK_ID/rewrite/$CHAPTER" \
  -H "Content-Type: application/json" \
  -d '{"brief":"回滚后重写本章，保留主线冲突"}' | jq .
```

确认：

```bash
curl -s "$BASE_URL/api/books/$BOOK_ID/chapter-runs?chapter=$CHAPTER&limit=10" | jq .
```

返回体中的 `rolledBackTo` 即本次回滚目标章号。

## 5. Chaos 演练清单

| 场景 | 命令 | 通过信号 |
| --- | --- | --- |
| 模型超时 | `pnpm --filter @actalk/inkos-studio test -- src/api/server.test.ts -t "chaos.*time"` | 任务变为 `failed`，错误含 `timed out`，章节原文不变 |
| snapshot 损坏 | `pnpm --filter @actalk/inkos-studio test -- src/api/server.test.ts -t "chaos.*snapshot"` | 服务可启动，损坏快照被有效 JSON 覆盖 |
| 预算耗尽中断 | `pnpm --filter @actalk/inkos-studio test -- src/api/server.test.ts -t "chaos.*budget"` | 返回策略阻断，`audit/revise` 不执行，章节原文不变 |

一键执行全部 chaos：

```bash
pnpm --filter @actalk/inkos-studio test -- src/api/server.test.ts -t chaos
```
