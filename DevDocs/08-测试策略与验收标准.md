# 测试策略与验收标准

## 1. 测试目标

- 核心创建流程可稳定完成。
- 双模式流程行为一致且可预测。
- 章节连续性提示逻辑可复现。

## 2. 测试分层

## 2.1 单元测试

后端：
- brief schema 校验
- brief normalize 逻辑
- create-flow 状态机转移
- 错误码映射

前端：
- 模式切换组件
- 摘要编辑组件
- 补问卡片交互
- 创建按钮可用性条件

## 2.2 集成测试

- `POST /normalize -> /confirm -> /create-status` 全链路
- 创建失败重试
- 同名书籍冲突 409
- brief 持久化文件写入验证

## 2.3 端到端测试（E2E）

场景 S1：新手快速创建
1. 选择快速模式
2. 输入自然语言
3. 确认摘要
4. 创建成功并跳转书籍详情

场景 S2：专业模式创建
1. 完成四步输入
2. 预览基础文件
3. 确认创建

场景 S3：异常恢复
1. 模型异常导致 normalize 失败
2. 用户手动编辑并继续创建

## 3. 回归测试范围

每次发布必须回归：
- 现有 `BookCreate` 基础流程（旧流程）
- 章节写作接口（write/draft/revise/audit）
- 日志与 SSE 不可中断

## 4. 数据测试集建议

准备 6 组标准输入：
1. 极简一句话输入
2. 超长详细输入
3. 缺失主角信息
4. 缺失结局方向
5. 中英混合输入
6. 含敏感词输入

用于验证 normalize 稳定性与补问逻辑。

## 5. 验收标准（MVP）

功能验收：
1. 快速模式可完成建书。
2. 专业模式可完成建书。
3. 摘要确认页可编辑并生效。
4. 创建失败有明确错误提示。

质量验收：
1. 核心接口成功率 >= 98%（测试环境）。
2. 前端主流程无阻断级 bug（P0/P1）。
3. E2E 主路径通过率 100%。

## 6. 自动化建议

- CI 必跑：
  - `pnpm -r test`
  - `pnpm -r typecheck`
- 针对新接口补充 API 契约快照测试。
- nightly 跑一次创建流程 E2E。

## 7. 缺陷分级

- P0：阻断创建/数据丢失
- P1：主流程严重错误但可绕过
- P2：体验缺陷/提示不清
- P3：样式与次要问题

发布门禁：
- 不允许存在 P0/P1 未修复缺陷。

## 8. 跨端与上架测试（新增）

- iOS：
  - TestFlight 核心路径全通过
  - 隐私权限与账号删除流程可用

- Android：
  - 主流机型兼容性
  - 前后台切换恢复

- Windows/macOS：
  - 安装升级卸载
  - 本地缓存与同步恢复

## 9. Agent 回归测试（新增）

- 每个任务由 Agent 自动执行：
  - 类型检查
  - 单元测试
  - 最小 E2E
  - 文档链接检查

- 人工只对高风险变更做抽检验收。
