# 跨端应用与上架方案

## 1. 目标平台

- iOS（Apple App Store）
- Android（Google Play 或国内分发）
- Windows（安装包或商店）
- macOS（App Store 或 DMG/PKG）

## 2. 技术路线建议

优先推荐：
- 业务前端保持 TypeScript 体系
- 移动端：React Native + Expo
- 桌面端：Tauri（轻量）或 Electron（生态成熟）
- 后端：统一 BFF/API，避免端侧直连复杂服务

## 3. 跨端能力边界

统一能力：
- 登录与账号
- 项目列表与同步
- 双模式创建
- 章节生成与审稿
- 中英文切换

端特有能力：
- 移动端：弱网重试、后台唤醒、推送通知
- 桌面端：批量导入导出、快捷键、窗口多任务

## 4. 上架必备清单（iOS重点）

1. App 隐私政策 URL
2. 服务条款 URL
3. 账号创建与删除能力
4. 数据收集说明（App Privacy）
5. 内容声明（AI 辅助创作）
6. 敏感内容处理与举报机制（若社区化）

## 5. 提审前质量门禁

- 崩溃率低于阈值
- 冷启动时间达标
- 核心流程（建书->写章->保存）无阻断
- 离线与弱网场景可恢复
- 国际化文案不缺失

## 6. 账号与同步

建议采用：
- 统一账号体系（邮箱/手机号/第三方登录）
- 设备会话管理（可远程下线）
- 文件级版本同步（含冲突提示）

## 7. 版本发布流程

1. `alpha`：内部 QA 与 Agent 回归
2. `beta`：小范围创作者试用
3. `rc`：提审候选
4. `ga`：正式发布

## 8. 回滚策略

- 客户端开关降级到旧创建流程
- 服务端保留 `/api/books/create` 兼容路径
- 同步异常时允许本地离线模式继续写作
